# Dynamic Programming — Динамическое программирование

## About

Динамическое программирование (DP) — метод решения задач, основанный на разбиении задачи на перекрывающиеся подзадачи. Результаты решения подзадач сохраняются (мемоизация или табуляция), чтобы избежать повторных вычислений.

## Принцип работы

1. Разбиение на подзадачи: исходная задача разбивается на подзадачи, которые могут быть решены независимо.
1. Мемоизация (или табуляция): результаты решения подзадач сохраняются в таблице, чтобы избежать повторного вычисления.
1. Объединение решений: решение исходной задачи строится на основе решений подзадач.

### Когда использовать

- Задача разбивается на перекрывающиеся подзадачи (одни и те же подзадачи решаются многократно).
- Задача обладает свойством оптимальной подструктуры — оптимальное решение задачи содержит оптимальные решения подзадач.

## Как распознать DP-задачу

Нужны два свойства одновременно.

Оптимальная подструктура
Ответ для задачи собирается из ответов подзадач.
Например, рюкзак: чтобы решить для i предметов и веса W, надо знать ответ для i-1 предметов и веса W (не берём) и W - weight[i] (берём).
Фибоначчи: Fib(n) = Fib(n-1) + Fib(n-2).
Не DP: бюджет с синергией проектов — выбор одного проекта влияет на доходность других, подзадачи не независимы.

Перекрывающиеся подзадачи
Одни и те же подзадачи считаются много раз.
Фибоначчи: fib(3) считается дважды, fib(2) — трижды. Без мемоизации O(2^n).
Не DP: быстрая сортировка (массив делится на непересекающиеся части), бинарный поиск (идём только в одну половину).

Типичные признаки

- Ищем минимум/максимум (цена, прибыль, путь).
- Считаем количество способов.
- Определяем, возможно ли (true/false).
- На каждом шаге выбор (взять/не взять, направо/налево).
- Жадность не работает, полный перебор слишком медленный.

## Пример: Задача о рюкзаке (0/1 Knapsack)

Описание: Дано множество предметов, каждый имеет вес и стоимость. Выбрать набор предметов для рюкзака ограниченной вместимости так, чтобы суммарная стоимость была максимальной.

Решение DP:

- Таблица `dp[i][w]` — максимальная стоимость для первых `i` предметов при вместимости `w`.
- Для каждого предмета выбор: взять или не взять.
- Сложность: O(n × W) времени, O(n × W) памяти, где n — число предметов, W — вместимость.

Когда использовать: ограниченная вместимость, каждый предмет можно взять не более одного раза.

```go
package main

import "fmt"

func knapsack(weights []int, values []int, capacity int) int {
 n := len(weights)
 dp := make([][]int, n+1)
 for i := range dp {
        dp[i] = make([]int, capacity+1)
    }

    for i := 1; i <= n; i++ {
        for w := 1; w <= capacity; w++ {
            if weights[i-1] <= w {
                dp[i][w] = max(dp[i-1][w], dp[i-1][w-weights[i-1]]+values[i-1])
            } else {
                dp[i][w] = dp[i-1][w]
            }
        }
    }

    return dp[n][capacity]
}

func max(a, b int) int {
    if a > b {
        return a
    }
    return b
}

func main() {
    weights := []int{2, 3, 4, 5}
    values := []int{3, 4, 5, 6}
    capacity := 5

    result := knapsack(weights, values, capacity)
    fmt.Printf("Максимальная стоимость: %d\n", result)
}
```

## Сравнение подходов

| Подход | Описание | Память | Типичная сложность |
| ------ | -------- | ------ | ------------------ |
| Top-Down (сверху вниз) | Рекурсия + мемоизация | O(число подзадач) + стек вызовов | Зависит от задачи |
| Bottom-Up (снизу вверх) | Итеративное заполнение таблицы | O(размер таблицы) | Зависит от задачи |

## Где используется

- Оптимизация: раскрой материалов, управление запасами, планирование производства
- Теория графов: кратчайшие пути (Флойд-Уоршелл, Беллман-Форд)
- Биоинформатика: выравнивание последовательностей (Needleman–Wunsch, Smith–Waterman)
- Ресурсы: задача о рюкзаке, распределение ресурсов
- Игры: минимакс, теорвер в покере
- Обработка текста: редакционное расстояние (Левенштейн), LCS
