# Boyer-Moore

## About

Алгоритм Бойера-Мура (Boyer-Moore) — один из самых эффективных алгоритмов поиска
подстроки на практике. Разработан Робертом Бойером и Джеймсом Муром в 1977 году.

Ключевая особенность — сравнение символов выполняется справа налево. Это позволяет
пропускать большие участки текста, делая алгоритм суб-линейным в лучшем случае
(O(n / m)).

## Основные характеристики

- **Временная сложность**:
  - Лучший случай: O(n / m) — суб-линейное время
  - Средний случай: O(n)
  - Худший случай: O(n × m)
- **Пространственная сложность**: O(σ), σ — размер алфавита (256 для ASCII)
- **Тип данных**: Строки (последовательности символов)

## Принцип работы

1. **Таблица плохих символов** (bad character table): для каждого символа алфавита
   запоминается его последняя позиция в паттерне.
1. **Поиск**: сравнение паттерна с текстом справа налево.
   При несовпадении:
   - Берём несовпавший символ текста.
   - По таблице определяем последнюю позицию этого символа в паттерне.
   - Сдвигаем паттерн так, чтобы эти символы совместились.
   - Минимальный сдвиг — 1 (если символ левее или отсутствует).

### Пример

```text
Текст:    ... D A B A B C A ...
Паттерн:    A B A B C A
                  ^ начало сравнения справа
              D не совпал с A
              D в pattern на позиции -1 (нет)
              Сдвиг: j - (-1) = 5 - (-1) = 6
Паттерн:          A B A B C A  (сдвиг на 6 позиций)
```

## Где используется

- **Текстовые редакторы**: поиск в больших файлах (на практике быстрее KMP)
- **Утилиты grep**: многие реализации используют вариации Бойера-Мура
- **Биоинформатика**: поиск длинных паттернов в геноме
- **Не рекомендуется**:
  - Для маленьких алфавитов (ДНК: A, C, G, T) — частые несовпадения, сдвиги малы
  - Когда критичен худший случай — KMP гарантирует O(n + m)

## Сравнение с аналогами

| Алгоритм | Лучший случай | Средний | Худший | Память |
|----------|---------------|---------|--------|--------|
| Наивный | O(n) | O(n × m) | O(n × m) | O(1) |
| KMP | O(n + m) | O(n + m) | O(n + m) | O(m) |
| Бойер-Мур | O(n / m) | O(n) | O(n × m) | O(σ) |

Бойер-Мур — лучший выбор для поиска в длинных текстах с большим алфавитом.
Он же — худший выбор для коротких паттернов в маленьком алфавите.
