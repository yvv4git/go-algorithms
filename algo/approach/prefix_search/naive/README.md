# Naive Search (Brute Force)

## About

Наивный алгоритм (brute force) — простейший алгоритм поиска подстроки в строке.
Последовательно проверяет каждую позицию текста на совпадение с паттерном.

Не использует предварительную обработку паттерна. Служит базовой точкой отсчёта
для сравнения с более эффективными алгоритмами (KMP, Рабин-Карп, Бойер-Мур).

## Основные характеристики

- **Временная сложность**: O(n × m), где n — длина текста, m — длина паттерна
- **Пространственная сложность**: O(1) (без учёта результатов)
- **Тип данных**: Строки (последовательности символов)
- **Свойство**: Не требует дополнительной памяти

## Принцип работы

1. Для каждой позиции i от 0 до n - m:
   - Сравнить text[i..i+m-1] с pattern[0..m-1] посимвольно.
   - Если все символы совпали — добавить i в результат.
1. Если совпадения не найдены — вернуть пустой массив.

### Пример

```text
Текст:    A B A B D A B A C D A B A B C A B A B
Паттерн:  A B A B C

Проверка:
i=0: A B A B D  →  D ≠ C  →  сдвиг
i=1: B A B D A  →  B ≠ A  →  сдвиг
...
i=10: A B A B C  →  совпадение!  →  позиция 10
```

## Когда и где использовать?

- **Короткие тексты**: когда n × m невелико
- **Образовательные цели**: демонстрация базового подхода к поиску подстрок
- **Прототипирование**: быстрая реализация без дополнительных структур
- **Не рекомендуется**:
  - Для длинных текстов (быстро деградирует до O(n × m))
  - Когда нужна гарантированная производительность
  - В production-коде с большими объёмами данных

## Сравнение с аналогами

| Алгоритм | Время (среднее) | Время (худшее) | Память |
|----------|-----------------|----------------|--------|
| Наивный | O(n × m) | O(n × m) | O(1) |
| KMP | O(n + m) | O(n + m) | O(m) |
| Рабин-Карп | O(n + m) | O(n × m) | O(1) |
| Бойер-Мур | O(n / m) | O(n × m) | O(σ) |
| Ахо-Корасик | O(n + m + k) | O(n + m + k) | O(m) |

Наивный алгоритм проигрывает всем современным алгоритмам в производительности,
но выигрывает в простоте реализации и нулевом расходе памяти.
