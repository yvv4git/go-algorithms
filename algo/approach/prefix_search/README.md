# Prefix Search — Поиск подстрок и префиксов

## About

Раздел содержит реализации алгоритмов поиска подстрок (паттернов) в тексте,
а также структуры данных для эффективного поиска по префиксам.

Задача поиска подстроки — одна из фундаментальных в информатике. Наивный алгоритм
работает за O(n × m), где n — длина текста, m — длина паттерна. Алгоритмы в этом
разделе улучшают эту оценку до линейной O(n + m) за счёт предварительной обработки
паттерна (или множества паттернов).

## Алгоритмы

### 1. KMP (Кнут-Моррис-Пратт) — `kmp/kmp_v1.go`

Поиск **одного** паттерна в тексте за O(n + m).

- **Префикс-функция (LPS)**: для каждого префикса паттерна вычисляется длина
  наибольшего собственного префикса, совпадающего с суффиксом.
- **Основная идея**: при несовпадении сдвигать паттерн не на 1 символ, а на
  максимально возможную величину, используя уже вычисленную LPS.
- **Сложность**: O(n + m) времени, O(m) памяти.
- **Когда использовать**: поиск одного паттерна в длинном тексте,
  когда важен детерминированный O(n).

### 2. Ахо-Корасик — `aho_corasick/ak_v1.go`

Поиск **множества** паттернов за один проход по тексту.

- **Бор (trie) + fail-ссылки**: все паттерны объединяются в префиксное дерево,
  для каждого узла вычисляется fail-ссылка (аналог LPS из KMP, но для дерева).
- **Основная идея**: проход по тексту без отката — при несовпадении переход
  по fail-ссылке, что позволяет найти все паттерны за O(n + m + k).
- **Сложность**: O(n + m + k) времени, O(m) памяти.
- **Когда использовать**: поиск множества паттернов (словарь ключевых слов,
  антивирусные сигнатуры, спам-фильтры).

### 3. Бор (Trie) — `bor_trie/bor_trie_v1.go`

Структура данных для хранения строк с поддержкой поиска по префиксу.

- **Префиксное дерево**: каждый путь от корня к листу соответствует строке.
  Общие префиксы хранятся один раз.
- **Операции**: Insert, Search, StartsWith — все за O(m).
- **Сложность**: O(m) на операцию, память O(суммарная длина всех строк).
- **Когда использовать**: автодополнение (autocomplete), проверка орфографии,
  IP-маршрутизация.

## Сравнение

| Алгоритм | Паттернов | Время | Память | Префиксный поиск |
| ---------- | ----------- | ------- | -------- | ------------------ |
| KMP | 1 | O(n + m) | O(m) | Нет |
| Ахо-Корасик | M (много) | O(n + m + k) | O(m) | Нет |
| Бор (Trie) | M (словарь) | O(m) на слово | O(словарь) | Да |

- **n** — длина текста
- **m** — длина паттерна (или суммарная длина всех паттернов)
- **k** — общее число найденных вхождений

## Где используется

- **Полнотекстовый поиск**: поиск ключевых слов в документах
- **Антивирусы и IDS/IPS**: обнаружение сигнатур в трафике
- **Автодополнение**: подсказки при вводе текста (Trie)
- **Компиляторы**: лексический анализ, разбиение на токены
- **Поиск в биоинформатике**: поиск паттернов в геномных последовательностях
- **Спам-фильтры**: поиск запрещённых слов в сообщениях (Ахо-Корасик)
- **Текстовые редакторы**: подсветка синтаксиса, поиск и замена
