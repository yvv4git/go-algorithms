# Bitap (Shift-OR)

## About

Алгоритм Bitap (также известный как Shift-OR) — это битовый алгоритм поиска подстроки,
использующий битовые маски и операции над машинным словом для параллельного сравнения.

Разработан Уди Манбером и Сунитом Ву в 1991 году. Является основой для `agrep` —
утилиты нечёткого поиска. Поддерживает поиск с ошибками (нечёткий поиск) с минимальными
изменениями.

## Основные характеристики

- **Временная сложность**: O(n), где n — длина текста (при m ≤ 64)
- **Пространственная сложность**: O(σ + m), σ — размер алфавита
- **Тип данных**: Строки (последовательности символов)
- **Ограничение**: m ≤ размер машинного слова (32 или 64 бита)

## Принцип работы

1. Для каждого символа алфавита строится битовая маска: 0 в позиции j означает,
   что символ совпадает с pattern[j]. Остальные биты — 1.
1. Текущее состояние R — битовая маска длины m: 0 в позиции j означает,
   что префикс pattern[0..j] совпадает с суффиксом обработанного текста.
1. Для каждого символа текста: R = (R \<< 1) | mask[ch].
1. Если старший бит R = 0 — паттерн найден на текущей позиции.

### Пример

```text
Паттерн: "ABAB" (m = 4)
R = 1111 (нет совпадений)

После 'A': R = (1111 << 1) | mask[A] = 1110 | mask[A]
После 'B': R = (1110 << 1) | mask[B] = 1100 | mask[B]
...
R = 0111 — совпадение первых трёх символов
R = 0000 — совпадение всех m символов
```

## Где используется

- **Нечёткий поиск**: `agrep`, поиск с опечатками (k ошибок)
- **Текстовые редакторы**: быстрый поиск для коротких паттернов
- **Встраиваемые системы**: простая реализация на маломощных устройствах
- **Не рекомендуется**:
  - Когда m > 64 (требуется эмуляция многокомпонентного состояния)
  - Для длинных паттернов — лучше KMP или Бойер-Мур

## Сравнение с аналогами

| Алгоритм | Время | Память | m ≤ 64 | Нечёткий |
|----------|-------|--------|--------|----------|
| Bitap | O(n) | O(σ) | Да | Да |
| KMP | O(n + m) | O(m) | — | Нет |
| Бойер-Мур | O(n/m) лучший | O(σ) | — | Нет |

Bitap — единственный алгоритм, который поддерживает нечёткий поиск (k различий)
без значительного увеличения времени работы. Для точного поиска уступает Бойеру-Муру
на практике.
