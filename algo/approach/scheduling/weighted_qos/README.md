# Weighted QoS

## About

Пакет реализует взвешенное распределение выбора приоритетов (Weighted QoS) для
решения проблемы голодания (starvation) при обработке задач краулинга.

При использовании обычной сортировки по `priority DESC` задачи с высоким
приоритетом вытесняют низкоприоритетные, которые могут никогда не получить
процессорное время. Weighted QoS гарантирует, что каждый приоритет получает
долю вызовов, пропорциональную его весу, независимо от наличия более
приоритетных задач.

**Формула веса:** `w(p) = p - minPriority + 1`

По умолчанию minPriority = 0, maxPriority = 10:

- priority 0 → вес 1
- priority 1 → вес 2
- ...
- priority 10 → вес 11

Общий вес: 66.

## Algorithm

Пакет предоставляет две стратегии выбора приоритета на основе весов:

### Random (PickRandom)

Генерируется случайное число `r ∈ [0, totalWeight)`, затем выполняется
линейный проход по массиву cumulative weights. Приоритет выбирается по
первому индексу, где `r < cumWeights[i]`.

Вероятность выбора priority = `weight(p) / totalWeight`.

- Time complexity: O(N) — линейный проход по cumulative weights
- Space complexity: O(N) — массив cumulative weights, где N = количество уровней приоритета

### Cyclic (PickCyclic)

На этапе инициализации строится развёрнутый список, где каждый priority
повторён `weight(p)` раз. Порядок — от большего priority к меньшему:
`[10×11, 9×10, ..., 1×2, 0×1]`. Всего 66 элементов.

Счётчик `atomic.Int64` инкрементируется при каждом вызове, результат:
`priorities[counter % len(priorities)]`.

За полный цикл (66 вызовов) каждый priority выбирается ровно `weight(p)` раз.
Это даёт детерминированное и строго равномерное распределение.

- Time complexity: O(1) — доступ по индексу
- Space complexity: O(W) — развёрнутый список, где W = totalWeight (сумма весов всех приоритетов)
