# Algorithm Aho-Corasick

## About

Алгоритм Ахо-Корасик — это алгоритм поиска множества подстрок (паттернов) в тексте,
построенный на основе бора (trie) с fail-ссылками. Разработан Альфредом Ахо и Маргарет Корасик в 1975 году.

Позволяет за один проход по тексту найти все вхождения любого из заданных паттернов,
что делает его оптимальным для задач мультипаттернного поиска.

## Основные характеристики

- **Временная сложность**: O(n + m + k), где n — длина текста, m — суммарная длина паттернов, k — число вхождений
- **Пространственная сложность**: O(m), где m — суммарная длина паттернов
- **Тип данных**: Строки (последовательности символов)
- **Принцип построения**: Бор (trie) + fail-ссылки (аналог LPS из KMP)

## Принцип работы

1. **Построение бора** — из всех паттернов строится префиксное дерево (trie).
   Каждый узел соответствует префиксу одного или нескольких паттернов.
   Узлы, в которых заканчивается паттерн, помечаются как терминальные.

1. **Построение fail-ссылок** — для каждого узла определяется fail-ссылка:
   указатель на самый длинный собственный суффикс текущей строки, который
   одновременно является префиксом какого-либо паттерна. Это аналог префикс-функции
   из KMP, но для бора. Строится с помощью BFS (обхода в ширину).

1. **Поиск по тексту** — проход по тексту слева направо. Текущий узел перемещается
   по бору в соответствии с символами текста. При несовпадении — переход по fail-ссылке.
   При посещении каждого узла проверяется, не заканчивается ли в нём какой-либо паттерн
   (включая паттерны, доступные по цепочке fail-ссылок).

## Визуализация

### Бор для паттернов ["he", "she", "his", "hers"]

```text
         root
        /    \
      h       s
     / \      |
    e   i     h
    |   |     |
   (r)  s    e
    |   |    |
   (s) ( )  ( )
```

- `(r)` — терминальные узлы с паттернами

### Fail-ссылки (пунктир)

- 'she' → 'he' (суффикс "he" — префикс "he")
- 'his' → 'is' (если бы 'is' был паттерном)
- 'hers' → 'he' → 'ers' (суффикс "he" — префикс "he")

## Когда и где использовать?

- **Мультипаттернный поиск**: поиск множества запрещённых слов в тексте
- **Антивирусные сигнатуры**: обнаружение известных сигнатур в потоке данных
- **Спам-фильтры**: поиск множества ключевых слов в письмах
- **Синтаксический анализ**: лексический анализ в компиляторах (flex-подобные инструменты)
- **Поиск подстрок в реальном времени**: IDS/IPS системы, мониторинг трафика
- **Не рекомендуется**:
  - Когда нужно найти только один паттерн (лучше использовать KMP)
  - Паттерны очень короткие (1-2 символа) — накладные расходы на построение бора не оправданы
  - Когда число паттернов мало (< 3) и текст короткий

## Сравнение с аналогами

| Алгоритм | Поиск одного паттерна | Поиск M паттернов | Сложность |
|----------|----------------------|-------------------|-----------|
| KMP | O(n + m) | O((n + m) × M) | O(n + m) |
| Ахо-Корасик | O(n + m) | O(n + m + k) | O(n + m) |
| Regex (NFA/DFA) | O(n × m) | O(n × m) | Экспонента в худшем |

Ахо-Корасик даёт выигрыш при большом количестве паттернов, так как все они обрабатываются
за один проход по тексту без необходимости повторного сканирования.
