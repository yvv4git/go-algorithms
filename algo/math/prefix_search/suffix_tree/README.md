# Suffix Tree (Суффиксное дерево)

## About

Суффиксное дерево — это сжатый бор (trie) всех суффиксов строки. Каждый путь
от корня к листу соответствует суффиксу. В отличие от обычного бора, рёбра могут
содержать подстроки (не только один символ), что делает структуру компактной.

Построение наивным алгоритмом — O(n²). Алгоритм Укконена строит суффиксное дерево
за линейное время O(n). Поиск подстроки выполняется за O(m).

## Основные характеристики

- **Временная сложность**:
  - Построение: O(n²) — наивное (O(n) — алгоритм Укконена)
  - Поиск: O(m), где m — длина паттерна
- **Пространственная сложность**: O(n) — количество узлов (с большими константами)
- **Тип данных**: Строки (последовательности символов)

## Принцип работы

1. Каждый суффикс строки вставляется в сжатый бор.
1. При вставке нового суффикса существующее ребро может быть расщеплено:
   если совпадает только часть ребра, оно делится на два ребра в точке расхождения.
1. Поиск — спуск по дереву по символам паттерна.
   Если паттерн прочитан — собираем все суффиксные ID в поддереве.

### Пример

```text
Строка: "ABCAB"
Суффиксы: "ABCAB", "BCAB", "CAB", "AB", "B"

Суффиксное дерево (упрощённо):
         root
        /  |  \
      AB  BCAB  CAB
      /    |
    CAB    AB
    |
   (лист)
```

## Где используется

- **Биоинформатика**: поиск длинных паттернов в геномных последовательностях
- **Поиск наибольшей общей подстроки**: две строки — O(n + m)
- **Поиск повторов**: поиск всех повторяющихся подстрок
- **Индексация текста**: полнотекстовый поиск с максимальной скоростью
- **Не рекомендуется**:
  - Когда важна компактность в памяти (суффиксный массив компактнее)
  - Для коротких строк — накладные расходы не оправданы
  - Когда нужна простота реализации (алгоритм Укконена сложен)

## Сравнение с аналогами

| Структура | Построение | Поиск | Память |
|-----------|------------|-------|--------|
| Суффиксное дерево | O(n) — Укконен | O(m) | O(n) большие константы |
| Суффиксный массив | O(n) — SA-IS | O(m log n) | O(n) |
| Бор (Trie) | O(n²) | O(m) | O(n²) |

Суффиксное дерево — самая мощная структура для поиска подстрок: O(m) поиск,
возможность искать произвольные подстроки (не только префиксы), поддержка
множества дополнительных запросов (LCA, количество вхождений, наибольший
общий префикс). Плата — сложность реализации и большие константы по памяти.
