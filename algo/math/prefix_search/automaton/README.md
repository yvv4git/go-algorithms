# Finite Automaton Matcher

## About

Алгоритм поиска подстроки с помощью конечного автомата (ДКА). По паттерну строится
детерминированный конечный автомат с m + 1 состоянием, после чего поиск по тексту
выполняется за O(n) без единого отката.

Состояние автомата q (0 ≤ q ≤ m) означает длину наибольшего префикса паттерна,
который является суффиксом прочитанной части текста. Как только q достигает m —
паттерн найден.

## Основные характеристики

- **Временная сложность**: O(n + m × σ), где n — длина текста, m — длина паттерна,
  σ — размер алфавита (256 для ASCII)
- **Пространственная сложность**: O(m × σ) — таблица переходов
- **Тип данных**: Строки (последовательности символов)
- **Свойство**: Ни одного лишнего сравнения — каждый символ текста обрабатывается ровно 1 раз

## Принцип работы

1. **Построение префикс-функции** π для паттерна (как в KMP).
1. **Построение таблицы переходов** δ(q, ch) для q = 0..m и всех ch алфавита:
   - Если ch = pattern[q], то δ(q, ch) = q + 1.
   - Иначе δ(q, ch) = δ(π[q−1], ch) (рекурсивно через префикс-функцию).
1. **Поиск**: проход по тексту, на каждом шаге q = δ(q, text[i]).
   Если q = m — найдено вхождение на позиции i − m + 1.

### Пример таблицы переходов для pattern = "AB"

```text
Состояние | ... 'A' 'B' 'C' ...
--------------------------------
0         | ...  1   0   0  ...
1         | ...  1   2   0  ...
2         | ...  1   0   0  ...
```

## Где используется

- **Лексический анализ**: регулярные выражения в компиляторах
- **Текстовые редакторы**: поиск с подсветкой
- **Сетевые фильтры**: поиск сигнатур на аппаратном уровне
- **Не рекомендуется**:
  - Когда алфавит большой (Unicode) — размер таблицы O(m × σ) становится огромным
  - Для коротких паттернов — накладные расходы на построение автомата не оправданы

## Сравнение с аналогами

| Алгоритм | Поиск | Предобработка | Память |
|----------|-------|---------------|--------|
| KMP | O(n) | O(m) | O(m) |
| Конечный автомат | O(n) | O(m × σ) | O(m × σ) |
| Ахо-Корасик | O(n) | O(m × σ) | O(m × σ) |

Конечный автомат — единственный алгоритм, где каждый символ текста обрабатывается
за строго O(1) (константное время, без циклов while). Это делает его предпочтительным
для аппаратной реализации (FPGA, ASIC).
