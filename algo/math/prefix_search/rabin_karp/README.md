# Rabin-Karp

## About

Алгоритм Рабина-Карпа (Rabin-Karp) — это алгоритм поиска подстроки в тексте,
основанный на хешировании. Разработан Ричардом Карпом и Майклом Рабином в 1987 году.

Ключевая идея — использовать кольцевой (rolling) хеш: хеш очередной подстроки текста
вычисляется из хеша предыдущей подстроки за O(1), без пересчёта с нуля.

## Основные характеристики

- **Временная сложность**:
  - Средний случай: O(n + m)
  - Худший случай: O(n × m) — при большом числе коллизий хеша
- **Пространственная сложность**: O(1) (без учёта результатов)
- **Тип данных**: Строки (последовательности символов)
- **Основа**: Хеширование + кольцевой хеш

## Принцип работы

1. Вычисляется хеш паттерна и хеш первой подстроки текста длины m.
1. Для каждой позиции i в тексте:
   - Если хеши совпадают — выполняется посимвольное сравнение для подтверждения.
   - Если не совпадают — хеш следующей подстроки вычисляется через кольцевой хеш:
     - Удаляется вклад левого символа: `hash = hash - text[i] × base^(m-1)`
     - Добавляется вклад нового символа: `hash = hash × base + text[i+m]`
1. Повтор до конца текста.

### Выбор хеш-функции

```text
hash = (c[0] × base^(m-1) + c[1] × base^(m-2) + ... + c[m-1]) mod prime
```

- **base** — основание (обычно 256 для ASCII или 26 для латиницы)
- **prime** — большое простое число (101, 9973, 10^9+7) для уменьшения коллизий

## Когда и где использовать?

- **Поиск нескольких паттернов**: можно расширить — хранить хеши всех паттернов
  в хеш-таблице и проверять за O(1) при совпадении хеша подстроки
- **Поиск с перекрытиями**: хорошо работает, когда паттерны одинаковой длины
- **Биоинформатика**: поиск коротких паттернов в геномных последовательностях
- **Не рекомендуется**:
  - Паттерны с длиной > текста (тривиальный случай)
  - Когда критичен худший случай O(n×m) (лучше KMP)
  - Когда алфавит очень мал (частые коллизии хеша)

## Сравнение с аналогами

| Алгоритм | Время (среднее) | Время (худшее) | Память | Проходов |
| ---------- | ----------------- | ---------------- | -------- | ---------- |
| Наивный | O(n × m) | O(n × m) | O(1) | 1 |
| KMP | O(n + m) | O(n + m) | O(m) | 1 |
| Рабин-Карп | O(n + m) | O(n × m) | O(1) | 1 |
| Ахо-Корасик | O(n + m + k) | O(n + m + k) | O(m) | 1 |

Rabin-Karp выигрывает у KMP и Наивного при поиске нескольких паттернов одинаковой
длины (хеши всех паттернов проверяются за O(1)). Проигрыш — недетерминированный
худший случай при коллизиях.
