# Suffix Array (Суффиксный массив)

## About

Суффиксный массив — это массив индексов всех суффиксов строки, отсортированных
в лексикографическом порядке. Разработан Уди Манбером и Джином Майерсом в 1990 году
как более компактная альтернатива суффиксному дереву.

Любая подстрока исходной строки является префиксом какого-либо суффикса, поэтому
поиск сводится к бинарному поиску по суффиксному массиву за O(m log n).

## Основные характеристики

- **Временная сложность**:
  - Построение: O(n² log n) — наивное (O(n log n) с удвоением, O(n) — SA-IS)
  - Поиск: O(m × log n), где m — длина паттерна, n — длина текста
- **Пространственная сложность**: O(n) для массива индексов
- **Тип данных**: Строки (последовательности символов)

## Принцип работы

1. Собираются все суффиксы строки (индекс + строка).
1. Суффиксы сортируются лексикографически.
1. Поиск паттерна — бинарным поиском по отсортированным суффиксам:
   - Находим левую границу (первый суффикс ≥ pattern).
   - Находим правую границу (первый суффикс > pattern).
   - Все суффиксы между ними начинаются с pattern.

### Пример

```text
Строка: "ABABDABACDABABCABAB"
SA:      17 15 10  0  5 12  2  7 18 16 11  1  6 13  3 14  8  9  4
Индекс:  суффикс (отсортирован)
  10:    "ABABCABAB"          ← начало паттерна
   ...
   4:    "DABACDABABCABAB"
```

## Где используется

- **Поиск подстрок**: быстрее суффиксного дерева по памяти
- **Сжатие данных**: BWT (Burrows-Wheeler Transform) на основе суффиксного массива
- **Биоинформатика**: поиск повторов, поиск паттернов в геноме
- **Индексация текста**: полнотекстовый поиск
- **Не рекомендуется**:
  - Для коротких строк — проще и быстрее KMP или Рабин-Карп
  - Когда нужна вставка/удаление (суффиксный массив статический)

## Сравнение с аналогами

| Структура | Построение | Поиск | Память |
|-----------|------------|-------|--------|
| Суффиксный массив | O(n) — SA-IS | O(m log n) | O(n) |
| Суффиксное дерево | O(n) — Укконен | O(m) | O(n) большие константы |
| Бор (Trie) | O(n²) | O(m) | O(n²) |

Суффиксный массив занимает меньше памяти, чем суффиксное дерево, но поиск медленнее
(O(m log n) против O(m)). В комбинации с массивом LCP даёт полную мощность
суффиксного дерева при меньших накладных расходах.
