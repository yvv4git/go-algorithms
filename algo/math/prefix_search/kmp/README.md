# KMP (Knuth-Morris-Pratt)

## About

Алгоритм Кнута-Морриса-Пратта (KMP) — это алгоритм поиска подстроки в строке,
использующий префикс-функцию (LPS) для избежания повторных сравнений.

Разработан Дональдом Кнутом, Джеймсом Моррисом и Воном Праттом в 1977 году.
Гарантирует линейное время O(n + m) в худшем случае, в отличие от наивного O(n × m).

## Основные характеристики

- **Временная сложность**: O(n + m), где n — длина текста, m — длина паттерна
- **Пространственная сложность**: O(m) — хранение LPS-массива
- **Тип данных**: Строки (последовательности символов)
- **Свойство**: Детерминированный, без откатов по тексту

## Принцип работы

1. **Префикс-функция (LPS)**: вычисляется массив π, где π[i] — длина наибольшего
   собственного префикса pattern[0..i], совпадающего с её суффиксом.

1. **Поиск**: проход по тексту слева направо. При несовпадении text[i] и pattern[j]
   паттерн сдвигается не на 1, а на j - π[j-1] позиций, используя уже сопоставленную
   информацию.

### Пример LPS

```text
Паттерн:  A B A B C A B A B
LPS:      0 0 1 2 0 1 2 3 4
```

## Визуализация

### Фаза 1: построение LPS

```text
i=1, j=0:  A ≠ B  →  π[1]=0
i=2, j=0:  A = A  →  π[2]=1, j=1
i=3, j=1:  B = B  →  π[3]=2, j=2
i=4, j=2:  A ≠ C  →  j=π[1]=0 → A ≠ C → π[4]=0
...
```

### Фаза 2: поиск

```text
Текст:    A B A B D A B A C D A B A B C A B A B
Паттерн:  A B A B C A B A B
                        ^ несовпадение D≠C
          Сдвиг по π[3]=2: j = π[3] = 2
Паттерн:      A B A B C A B A B
```

## Когда и где использовать?

- **Текстовые редакторы**: поиск и замена в длинных файлах
- **Биоинформатика**: поиск паттернов в геномных последовательностях
- **Компиляторы**: лексический анализ
- **Сетевой мониторинг**: поиск сигнатур в потоке данных
- **Не рекомендуется**:
  - Для коротких текстов (наивный алгоритм будет быстрее из-за меньших накладных расходов)
  - Когда нужен поиск множества паттернов (лучше Ахо-Корасик)

## Сравнение с аналогами

| Алгоритм | Время | Память | Паттернов |
|----------|-------|--------|-----------|
| Наивный | O(n × m) | O(1) | 1 |
| KMP | O(n + m) | O(m) | 1 |
| Рабин-Карп | O(n + m) среднее | O(1) | 1+ |
| Бойер-Мур | O(n/m) лучший | O(σ) | 1 |
| Ахо-Корасик | O(n + m + k) | O(m) | M |

KMP — лучший выбор, когда важен гарантированно линейный худший случай
и паттерн не очень короткий. Проигрывает Бойеру-Муру на практике в среднем,
но не деградирует до O(n × m).
