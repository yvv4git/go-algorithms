# Bor Trie (Prefix Tree)

## About

Бор (trie, префиксное дерево) — это структура данных для хранения множества строк,
где каждый узел соответствует одному символу. Все потомки узла имеют общий префикс —
строку, полученную при движении от корня к этому узлу.

Ключевая идея: общие префиксы строк хранятся в памяти один раз, что даёт эффективный
поиск по префиксам — быстрее, чем в хеш-таблицах или сбалансированных деревьях.

## Основные характеристики

- **Временная сложность**:
  - Вставка: O(m), где m — длина строки
  - Поиск слова: O(m)
  - Поиск префикса: O(m)
- **Пространственная сложность**: O(m × n) в худшем случае, где n — число строк, m — длина
  (зависит от того, насколько строки разделяют общие префиксы)
- **Устойчивость строк**: Порядок вставки не влияет на структуру
- **Алфавит**: Обычно ASCII / Unicode (доступен посимвольный обход)

## Принцип работы

1. **Вставка** — начиная с корня, для каждого символа слова создаётся или находится
   соответствующий дочерний узел. Последний узел помечается как конец слова (`isEnd = true`).

1. **Поиск слова** — обход от корня по символам. Если на каком-то шаге ребро отсутствует —
   слова нет. Если все символы пройдены, проверяется флаг `isEnd`.

1. **Поиск префикса** — обход от корня по символам. Если все символы пройдены —
   префикс существует (независимо от `isEnd`).

## Визуализация

### Бор для слов ["hello", "hell", "high", "hill", "world", "worlds"]

```text
         root
        /    \
      h       w
      |       |
      e       o
      |       |
      l       r
     / \      |
    l   i     l
    |  / \    |
    o g   l   d
    | |   |   |
   ( )()  () (s)
               |
              ( )
```

- Обычный узел — промежуточный символ
- `( )` — терминальный узел (isEnd = true)

Обратите внимание: слово `"hell"` не требует создания новых узлов после `"hel"`
(так как `"hello"` уже создало узлы `h → e → l`). Узел `l` (3-й уровень) помечен
как `isEnd = true`, так как на нём заканчивается слово `"hell"`.

## Когда и где использовать?

- **Автодополнение (autocomplete)**: быстрый поиск слов по набранному префиксу
- **Проверка орфографии**: словарь для проверки наличия слова
- **IP-маршрутизация (radix tree)**: поиск наиболее длинного префикса среди маршрутов
- **Лексический анализ**: разбиение текста на токены
- **Сжатие данных**: префиксное кодирование
- **Не рекомендуется**:
  - Когда алфавит очень большой (символы Unicode целиком) — раздувание памяти
  - Когда строк короткие, а их мало — проще использовать хеш-таблицу
  - Когда нужен поиск подстрок (не префиксов) — лучше использовать суффиксное дерево

## Сравнение с аналогами

| Структура | Поиск слова | Поиск префикса | Память |
| ----------- | ------------- | ---------------- | -------- |
| Хеш-таблица | O(m) | O(n × m) | O(n × m) |
| Бор | O(m) | O(m) | O(m × n) нестрого |
| Сбалансированное дерево | O(m × log n) | O(m × log n) | O(n × m) |
| Суффиксное дерево | O(m) | O(m) | O(n²) |

Бор — лучший выбор для задач, где критичен поиск по префиксу (автодополнение,
проверка префиксов). В отличие от хеш-таблицы, бор не требует перебора всех ключей
для поиска по префиксу.
