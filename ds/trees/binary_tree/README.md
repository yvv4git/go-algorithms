# Binary tree


## Информация
Бинарное дерево (Binary Tree) - это дерево, в котором у каждого узла не более двух дочерних элементов, 
обычно называемых левым и правым дочерним элементами. В бинарном дереве каждый узел может иметь от 0 до 2 дочерних узлов.

Бинарное дерево поиска (Binary Search Tree) - это бинарное дерево, в котором для всех узлов, лежащих в левом поддереве, 
значения этих узлов меньше, чем значение корня, а для всех узлов, лежащих в правом поддереве, значения этих узлов больше, чем значение корня.


## Отличие от BST
Отличия:
1. В бинарном дереве поиска значения узлов упорядочены, тогда как в обычном бинарном дереве этого не происходит.
2. В бинарном дереве поиска операции поиска, вставки и удаления имеют логарифмическую сложность, 
тогда как в обычном бинарном дереве эти операции могут иметь линейную сложность.
3. Бинарное дерево поиска может быть несбалансированным, тогда как бинарное дерево обычно является сбалансированным.


## Порядок обхода
В бинарном дереве существует три основных метода обхода:
1. Inorder (синтаксический, инфиксный): Вы посещаете левое поддерево, потом корень, и затем правое поддерево.
2. Preorder (префиксный): Вы посещаете корень, затем левое поддерево, и затем правое поддерево.
3. Postorder (постфиксный): Вы посещаете левое поддерево, затем правое поддерево, и затем корень.

Если мы используем алгоритм DFS(Depth-First Search) для обхода дерева, то может быть три варианта DFS:
1. Preorder DFS (Root, Left, Right)
2. Inorder DFS (Left, Root, Right)
3. Postorder DFS (Left, Right, Root)


## Где использовать
- используются для хранения данных, чтобы обеспечить быстрый поиск, вставку и удаление элементов с временной сложностью O(log n) в среднем случае.

## Визуализация

       / \
      2   3
     / \   / \
    4   5 6   7

  8   9


## Визуализация алгоритма Preorder DFS
### Preorder Traversal (корень, левое, правое)
Порядок обхода: 1 -> 2 -> 4 -> 8 -> 9 -> 5 -> 3 -> 6 -> 7

### Пошаговая работа Preorder DFS
- Шаг 1: Посетить корень 1. Порядок: [1]
- Шаг 2: Перейти к левому поддереву, посетить 2. Порядок: [1, 2]
- Шаг 3: Из 2 перейти к левому 4. Порядок: [1, 2, 4]
- Шаг 4: Из 4 перейти к левому 8. Порядок: [1, 2, 4, 8]
- Шаг 5: Из 8 вернуться к 4, перейти к правому 9. Порядок: [1, 2, 4, 8, 9]
- Шаг 6: Из 9 вернуться к 4, затем к 2, перейти к правому 5. Порядок: [1, 2, 4, 8, 9, 5]
- Шаг 7: Из 5 вернуться к 2, затем к 1. Перейти к правому поддереву 3. Порядок: [1, 2, 4, 8, 9, 5, 3]
- Шаг 8: Из 3 перейти к левому 6. Порядок: [1, 2, 4, 8, 9, 5, 3, 6]
- Шаг 9: Из 6 вернуться к 3, перейти к правому 7. Порядок: [1, 2, 4, 8, 9, 5, 3, 6, 7]

Алгоритм завершен. Финальный порядок: 1 -> 2 -> 4 -> 8 -> 9 -> 5 -> 3 -> 6 -> 7



## Визуализация алгоритма Inorder DFS

- Шаг 6: Из 9 вернуться к 4, затем к 2, посетить 2. Порядок: [8, 4, 9, 2]
- Шаг 7: Из 2 перейти к правому 5. Порядок: [8, 4, 9, 2, 5]
- Шаг 8: Из 5 вернуться к 2, затем к 1, посетить 1. Порядок: [8, 4, 9, 2, 5, 1]
- Шаг 9: Из 1 перейти к правому 3. Порядок: [8, 4, 9, 2, 5, 1, 3]
- Шаг 10: Из 3 перейти к левому 6. Порядок: [8, 4, 9, 2, 5, 1, 6]
- Шаг 11: Из 6 вернуться к 3, посетить 3. Порядок: [8, 4, 9, 2, 5, 1, 6, 3]
- Шаг 12: Из 3 перейти к правому 7. Порядок: [8, 4, 9, 2, 5, 1, 6, 3, 7]

Алгоритм завершен. Финальный порядок: 8 -> 4 -> 9 -> 2 -> 5 -> 1 -> 6 -> 3 -> 7


## Визуализация алгоритма Postorder DFS
### Postorder Traversal (левое, правое, корень)
Порядок обхода: 8 -> 9 -> 4 -> 5 -> 2 -> 6 -> 7 -> 3 -> 1

### Пошаговая работа Postorder DFS
- Шаг 1: Перейти к левому поддереву корня 1, посетить левое 2. Порядок: []
- Шаг 2: Из 2 перейти к левому 4. Порядок: []
- Шаг 3: Из 4 перейти к левому 8. Порядок: []
- Шаг 4: Из 8 вернуться к 4, перейти к правому 9. Порядок: []
- Шаг 5: Из 9 вернуться к 4, посетить 4. Порядок: [4]
- Шаг 6: Из 4 вернуться к 2, перейти к правому 5. Порядок: [4]
- Шаг 7: Из 5 вернуться к 2, посетить 2. Порядок: [4, 2]
- Шаг 8: Из 2 вернуться к 1, перейти к правому 3. Порядок: [4, 2]
- Шаг 9: Из 3 перейти к левому 6. Порядок: [4, 2]
- Шаг 10: Из 6 вернуться к 3, перейти к правому 7. Порядок: [4, 2]
- Шаг 11: Из 7 вернуться к 3, посетить 3. Порядок: [4, 2, 3]
- Шаг 12: Из 3 вернуться к 1, посетить 1. Порядок: [4, 2, 3, 1]

Алгоритм завершен. Финальный порядок: 8 -> 9 -> 4 -> 5 -> 2 -> 6 -> 7 -> 3 -> 1
